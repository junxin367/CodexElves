# 会话预热流光动效设计

## 目标

在 Codex 左侧会话列表中，仅对当前正在执行预热的会话标题显示蓝色流光。动效用于反馈后台预热状态，不改变预热调度、并发和 Owner 管理逻辑。

## 状态范围

- 等待队列中的会话不显示动效。
- worker 开始执行单条预热任务时显示动效。
- 单条任务成功、失败、转为前台或被运行时中断后移除动效。
- 当前并发为 4，因此最多同时显示 4 条流光。
- subagent、归档、临时、侧会话和运行中会话仍沿用现有过滤规则，不进入预热，也不显示动效。

## 展示方式

- 通过会话行的 `data-app-action-sidebar-thread-id` 与预热任务 `threadId` 精确匹配。
- 在 `[data-thread-title]` 等现有标题节点上增加状态属性，不修改标题文本和原始 DOM 层级。
- 使用伪元素叠加蓝色渐变文字光带，约 1.4 秒从左向右循环扫过。
- 原始文字始终保留，避免动画加载失败、主题切换或选中态下标题不可读。
- 动效不占用布局空间，不影响标题省略、悬停操作按钮和点击区域。
- `prefers-reduced-motion: reduce` 下取消循环动画，仅保留轻微的蓝色静态提示。

## 生命周期与 DOM 同步

维护一组“正在执行预热”的 thread ID：

1. `runCodexSessionPrewarmTask` 真正开始执行时加入 ID 并同步侧边栏。
2. `finally` 中无条件移除 ID 并清理侧边栏状态，覆盖成功和异常路径。
3. 侧边栏 React 重渲染或会话行重新挂载时，现有扫描流程根据 ID 集合重新应用状态。
4. 注入运行时刷新时清理旧节点属性，防止旧脚本留下静态流光。
5. 用户在预热期间点击该会话时，沿用现有前台提升逻辑；直到恢复任务完成才移除动效。

## 异常处理

- 找不到对应会话行时不报错；任务继续执行，后续侧边栏扫描会补挂动效。
- 标题节点不存在时不修改会话行，避免覆盖整行样式。
- DOM 节点被复用时，根据 thread ID 集合重新校正，清除不再预热的旧属性。
- 预热失败或 Promise 抛错时依靠 `finally` 清理，不允许动效残留。

## 验证

- 回归测试验证仅执行中的任务被标记，结束后清理。
- 验证等待任务不显示，最大同时标记数量与并发 4 一致。
- 验证侧边栏节点后挂载时可补充状态，节点复用后可清除旧状态。
- 验证失败路径、前台提升路径和减少动态效果样式。
- 执行 JavaScript 语法检查、`cdp_bridge` 测试、工作区检查、管理端类型检查和 `git diff --check`。
